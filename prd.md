# 🧭 Voyagé — AI 기반 여행 추천 서비스
**PRD v1.0 | 2025 | 초안**

---

## 0. 제품 개요

> **"일정을 선택하면 계절·날씨·기간을 고려해 최적의 해외 여행지를 추천하고, 환율·날씨·관광지 등 실전 정보를 즉시 제공하는 비로그인 여행 큐레이팅 서비스"**

Voyagé는 복잡한 회원가입 없이, 캘린더에서 출발·귀국 날짜를 선택하는 것만으로 여행 추천을 시작합니다. AI가 선택한 일정의 계절성, 체류 기간, 현지 기후를 분석해 국가별 여행지 후보를 제시하고, 사용자가 관심 목적지를 클릭하면 환율·날씨·주요 관광지 정보가 즉시 로드됩니다.

---

## 1. 핵심 기능 목록

### ✅ Must Have — 반드시 포함

#### 📅 일정 선택
- 에어비앤비 스타일 달력 UI — 체크인/체크아웃 날짜 선택
- 자동 계산: 박 수(nights), 여행 기간, 해당 계절 도출
- 최소 1박 ~ 최대 30박 범위 지원
- 모바일 반응형 달력 렌더링

#### 🌍 여행지 추천
- 일정 입력 → 국가별 여행지 카드 형태로 추천 (최소 6개국 이상)
- 추천 근거 표시 (예: "겨울에 따뜻한 동남아", "단기 여행에 최적")
- 계절·기간·날씨 3가지 기준 기반 AI 추천 로직
- 국가/지역 카테고리 필터 (아시아 / 유럽 / 아메리카 / 오세아니아)

#### 🔍 여행지 상세 정보
- 실시간 환율 정보 (원화 기준)
- 현지 평균 기온 및 날씨 예보
- 추천 관광지 목록 (최소 5곳)
- 간략한 여행 팁 (복장, 주의사항 등)

#### 🚀 기반 요건
- 비로그인 — 세션 기반 상태 관리 (쿠키 또는 localStorage)
- 페이지 전환 없는 SPA(Single Page Application) 흐름
- 로딩 스피너 및 스켈레톤 UI

---

### 💡 Nice to Have — 다음 버전 고려

#### ✨ 사용자 경험
- 즐겨찾기/찜하기 (로컬스토리지 저장)
- 여행지 비교 기능 (최대 3개 나란히 보기)
- 공유하기 버튼 (URL 파라미터로 일정+목적지 공유)
- 최근 조회 여행지 히스토리

#### 📊 콘텐츠 확장
- 항공권 가격 트렌드 (스카이스캐너 등 API 연동)
- 숙소 가격대 범위 표시
- 여행자 리뷰 요약 (Google Places 연동)
- 현지 공휴일·이벤트 캘린더
- 다국어 지원 (영어, 일본어 등)

## 2. 주요 사용자 시나리오

### 시나리오 01 — 겨울 단기 탈출 (직장인 2박 3일)

**페르소나:** 30대 직장인 · 연차 3일 확보 · 혼자 또는 커플

1. 사이트 접속 → 달력에서 12월 말 3일 선택
2. "겨울 단기 여행" 추천 카드 로드 (오키나와, 방콕, 홍콩 등)
3. 방콕 카드 클릭 → 환율(1THB = 38원), 평균기온 28°C 확인
4. 추천 관광지 5곳 + 복장 팁 확인 후 여행 계획 확정

---

### 시나리오 02 — 유럽 장기 배낭여행 (대학생 20일)

**페르소나:** 20대 대학생 · 여름방학 3주 · 유럽 첫 방문

1. 달력에서 7월 15일 ~ 8월 4일 (20박 21일) 선택
2. 유럽 탭 클릭 → 프라하, 리스본, 바르셀로나 등 추천 등장
3. 프라하 선택 → "여름 성수기 · 비교적 저렴한 물가" 태그 확인
4. 환율 및 관광지 정보 확인, 추가로 리스본도 탭 전환해 비교
5. 두 도시 정보 공유 URL 복사 후 친구에게 전송

---

### 시나리오 03 — 가족 휴양지 탐색 (부모 + 초등학생 자녀)

**페르소나:** 40대 부모 · 초등학생 자녀 2명 · 5박 6일 여름 가족 여행

1. 7월 말 5박 선택 → "가족 여행" 필터 태그 활성화
2. 발리, 다낭, 오키나와 — 가족 친화 여행지 우선 추천
3. 다낭 카드 클릭 → 아이 동반 팁(워터파크, 리조트 안전) 표시
4. 현지 기온 32°C · 우기 주의 경고 배너 확인
5. 여행 계획서 인쇄 버튼으로 PDF 저장

## 3. 기술 스택 추천

| 영역 | 추천 기술 | 선택 이유 |
|------|-----------|-----------|
| Frontend | React + TypeScript | 컴포넌트 재사용성, 타입 안정성. 달력 상태 관리에 적합 |
| 스타일링 | Tailwind CSS | 반응형 UI 빠른 구현. 모바일 달력 최적화 유리 |
| 달력 UI | react-day-picker | 에어비앤비 스타일 범위 선택 지원, 커스텀 가능 |
| 상태 관리 | Zustand | 경량 전역 상태. 일정 + 선택 여행지 관리에 충분 |
| AI 추천 | Anthropic Claude API | 여행지 추천 및 설명 생성 (claude-sonnet-4) |
| 환율 API | ExchangeRate-API | 무료 플랜으로 실시간 환율 제공 |
| 날씨 API | OpenWeatherMap API | 현재 날씨 + 5일 예보, 무료 플랜 가능 |
| 관광지 정보 | Google Places API | 명소 목록, 리뷰, 사진 제공 |
| 배포 | Vercel | GitHub 연동 무중단 배포. Next.js 미사용 시에도 지원 |
| 패키지 관리 | pnpm | 빠른 설치 속도, 디스크 효율. 팀 협업에 유리 |

### 아키텍처 흐름 요약

사용자가 달력에서 일정을 선택하면 프론트엔드에서 Zustand 스토어에 날짜 범위를 저장합니다. 해당 날짜의 계절/기간 정보를 Claude API에 전달해 여행지 추천 목록(JSON)을 받아 카드 UI로 렌더링합니다. 목적지 선택 시 ExchangeRate-API, OpenWeatherMap, Google Places API를 병렬로 호출해 상세 정보를 표시합니다. 모든 API 키는 서버사이드 환경 변수로 관리합니다.

## 4. 이번 버전에서 제외할 것들 (Out of Scope)

| 제외 항목 | 제외 이유 / 향후 계획 |
|-----------|----------------------|
| ⛔ 회원가입 / 로그인 | MVP 범위 불포함. 사용자 마찰 최소화가 핵심 전략. v2에서 소셜 로그인 고려 |
| ⛔ 항공권 · 숙소 예약 | 예약 시스템 연동은 별도 파트너십 필요. v2에서 제휴 딥링크 형태로 추가 |
| ⛔ 국내 여행지 추천 | 초기 버전은 해외 여행에 집중. 시장 반응 확인 후 국내 확장 |
| ⛔ 커뮤니티 · 후기 작성 | UGC 관리 비용 발생. 검증된 외부 리뷰(Google) 활용으로 대체 |
| ⛔ 다국어 UI 지원 | 한국어 단일 타겟으로 시작. 글로벌 수요 확인 후 i18n 도입 |
| ⛔ 여행 일정표 자동 생성 | Day-by-day 여행 플래너는 v3 기능. 이번 버전은 여행지 발견에 집중 |
| ⛔ 오프라인 모드 | PWA 오프라인 지원 미구현. 초기 구현 복잡도 절감 |
| ⛔ Push 알림 | 비로그인 서비스에 불필요. 로그인 도입 후 재검토 |
